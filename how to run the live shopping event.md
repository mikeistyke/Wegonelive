# How to Run the Live Shopping Event

## Purpose of This Runbook
This file is your **event operating system** from first customer contact to final close-out. It is designed to help you run a profitable live shopping event with low risk, strong buyer trust, and repeatable execution.

For your first events, this runbook assumes:
- Primary broadcast mode: **RTC host inside your app**
- Optional OBS/RTMP: **later phase**, only after stable rehearsals and confirmed ingest flow

---

## Core Business Model (Why This Works)
A live shopping event is not just a stream. It is a revenue system with five linked layers:

1. **Demand creation** (bring the right people at the right time)
2. **Trust acceleration** (prove product value live)
3. **Decision compression** (reduce buyer hesitation fast)
4. **Conversion mechanics** (clear “how to buy” in real time)
5. **Retention loop** (turn one-time buyers into repeat attendees)

If any one layer is weak, revenue suffers. Your workflow must intentionally operate all five.

## Event Workflow: Door Open → Door Close

## Phase 0: Offer + Audience Fit (Before Any Streaming)
Goal: Ensure the event has a clear money-making reason to exist.

### 0.1 Choose one event theme
Use one narrow lane per event:
- "Vintage Jackets Under $150"
- "Top 20 Collectibles Tonight"
- "Clearance Friday: Last Chance Lots"

### 0.2 Define conversion target
Pick one primary KPI:
- Units sold
- Gross revenue
- Paid buyers
- AOV (average order value)

### 0.3 Build lot stack by conversion logic
Order lots intentionally:
- **Hook lot** first (high perceived value, easy buy)
- **Margin lots** in middle
- **Hero lot** at peak attention
- **Fast-clearance lots** near end

### 0.4 Build your event promise
One sentence customers should remember:
- "Tonight we turn collector-grade items into same-night wins with verified condition live."

---

## Phase 1: Customer Acquisition Sprint (7–10 Days Before Event)
Goal: Fill the room with people most likely to buy.

Use your `customer-target-map` segments and run this cadence:

### 1.1 Daily outreach block (60 min)
- 10 new prospects
- 10 direct messages / emails
- 10 follow-ups
- 1 call booking target

### 1.2 Audience building assets
Create three assets:
1. Event landing post (date, theme, top 3 items)
2. 30-second teaser clip (or photo carousel)
3. "How to join + how to buy" one-pager

### 1.3 Reminder sequence
- T-72h: Main invite
- T-24h: "Set reminder" post
- T-2h: "Doors open soon"
- T-15m: "We’re live in 15"

### 1.4 Qualification question in outreach
Ask one filter question:
- "Which categories are you actively buying this month?"

This helps prioritize serious buyers and personalize lot order.

---

## Phase 2: Event Architecture (Operational Setup)
Goal: Reduce failure points and cognitive overload.

### 2.1 Assign operating roles (even if solo)
If solo, time-block role switching; if team, assign directly.

- **Host**: On-camera sales + pacing
- **Ops**: Lot transitions, timing, checks
- **Chat closer**: Answers and nudges buyer actions
- **Fallback operator**: Backup device/internet response

### 2.2 Technical reliability stack
For first events, prioritize resilience over complexity.

- One primary host device
- One backup host device (logged in, ready)
- Stable internet (wired preferred)
- Browser permissions pre-granted
- Avoid unnecessary apps/tabs

### 2.3 Broadcast mode policy
- Start with **RTC-only** until consistently stable
- One active publisher per channel
- No mid-show source switching unless rehearsed

---

## Phase 3: Rehearsal Protocol (Day Before + Day Of)
Goal: Remove unknowns before money is on the line.

### 3.1 Full dress rehearsal (30–45 min)
Run the exact show flow:
- Opening script
- Lot transitions
- Price callouts
- Bid handling
- Event close script

### 3.2 Failure drills
Practice 3 failure recoveries:
1. Audio drop
2. Video freeze
3. Host reconnect

### 3.3 Go/no-go checklist
Run this checklist 15 minutes before event:
- Host can start/stop broadcast
- Viewer receives stable audio/video
- Chat/Q&A response flow works
- Lot order visible and ready
- Backup device verified

If any critical check fails, delay 5–10 minutes and fix.

---

## Phase 4: Live Event Execution (Showtime)
Goal: Convert attention into transactions while preserving trust.

## 4.1 Opening the door (first 3 minutes)
Use this structure:
1. Welcome + event promise
2. Explain format (how bids/orders work)
3. Explain urgency (limited lots, live-only bonuses)
4. Start first lot quickly

Don’t spend too long on intro. Momentum beats perfection.

## 4.2 Lot cycle framework (repeat every lot)
For each lot, follow this exact sequence:

1. **Context**: What is it + why it matters
2. **Proof**: Show condition/features live
3. **Price anchor**: Expected value reference
4. **Offer**: Current ask / opening bid
5. **Urgency**: Time or quantity limit
6. **Close prompt**: Clear buyer action now
7. **Confirmation**: Winning buyer + next step

## 4.3 Communication discipline
- Short sentences, repeated buying instructions
- Frequent recap for new joiners
- Name buyers publicly (social proof)
- Keep cadence active (dead air kills conversions)

## 4.4 Conversion multipliers
- Bundle offer after every 2–3 lots
- "Add-on" products after hero lot
- Time-boxed bonuses (only during live)
- Fast yes/no CTAs (not vague prompts)

## 4.5 Runtime control metrics
Track these live signals:
- Active viewers
- Chat velocity
- Bid frequency
- Drop-off moments (what caused exits)

If engagement drops:
- Move to a stronger lot
- Ask one direct buying question
- Run a micro-offer (limited-time discount)

---

## Phase 5: Incident Response During Live
Goal: Recover in seconds, not minutes.

### 5.1 If audio/video glitches
- Acknowledge briefly: "Quick reset, stay with me."
- Stop publish
- Wait 3–5 seconds
- Restart publish
- Confirm recovery verbally

### 5.2 If primary host fails
- Move to backup host device
- Resume same channel/workflow
- Re-anchor audience with quick recap

### 5.3 If chat confusion appears
- Pause one lot
- Re-explain buying steps in 20 seconds
- Continue immediately

Rule: calm + clear + fast.

---

## Phase 6: Closing the Door (Final 5 Minutes)
Goal: maximize final conversions and set next-event momentum.

Use this close sequence:

1. "Last call" for final lots
2. Quick recap of top wins sold tonight
3. Final CTA for remaining inventory
4. Thank buyers by name/category
5. Preview next event date/theme (or "watch this channel")
6. End stream cleanly

### Closing script model
"That wraps tonight’s live. Thank you for showing up and buying with us. If you claimed a lot, watch for follow-up confirmation right after this. Next event details drop soon—same place, same energy. See you next time."

---

## Phase 7: Post-Event Monetization Loop (0–24 Hours)
Goal: turn one event into recurring revenue.

### 7.1 Immediate follow-up
- Confirm buyer orders quickly
- Resolve payment/fulfillment steps
- Send "missed tonight?" replay/highlight summary

### 7.2 Event debrief (same day)
Capture:
- Revenue
- Units sold
- Best-performing lots
- Drop-off points
- Top objections heard

### 7.3 Improvement plan (next event)
Write top 3 fixes only:
- One technical fix
- One conversion fix
- One offer fix

Small consistent upgrades beat major overhauls.

## Weekly Operating Rhythm (To Build Revenue Stability)

### Monday
- Offer design + lot sourcing

### Tuesday
- Prospect outreach + list growth

### Wednesday
- Content teasers + event page updates

### Thursday
- Follow-ups + buyer qualification

### Friday/Saturday
- Live event execution

### Sunday
- Review metrics + prepare next cycle

---

## First Event Non-Negotiables
- Keep stack simple (RTC-first)
- One active publisher at a time
- Scripted lot cycle (don’t improvise structure)
- Backup device ready
- Clear close + follow-up within 24 hours

---

## Simple Metrics Dashboard (Track Every Event)
- Attendees
- Peak concurrent viewers
- Buyers
- Conversion rate (buyers / attendees)
- Revenue
- AOV
- Sell-through rate (lots sold / lots shown)
- Repeat-buyer count (next event)

---

## Final Principle
For your first money-making events, don’t chase perfect tech. Chase reliable execution.

**Simple + consistent + trusted** beats complex + fragile every time.
