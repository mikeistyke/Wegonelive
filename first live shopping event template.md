# First Live Shopping Event Template

## 1) Pre-Live (60-30 min before)
- [ ] Use **RTC host flow only** for this first event (no OBS switching)
- [ ] Confirm internet is stable (prefer wired)
- [ ] Close heavy apps/tabs not needed for the event
- [ ] Open app and verify host can start/stop broadcast
- [ ] Verify mic and camera permissions in browser
- [ ] Confirm channel/item for event: `item-001` (or your chosen item)
- [ ] Verify audio is clear and not clipping
- [ ] Verify one backup device is ready and logged in

## 2) Final Checks (15 min before)
- [ ] Start a private test broadcast for 1-2 minutes
- [ ] Confirm viewer device receives video/audio
- [ ] Confirm bids/chat/QA still update during broadcast
- [ ] Confirm no duplicate host session is active
- [ ] Stop test broadcast cleanly

## 3) Go Live (Start of event)
- [ ] Start broadcast from host device
- [ ] Announce stream is live
- [ ] Keep one active publisher only
- [ ] Keep backup host device ready but idle

## 4) During Event
- [ ] Follow lot order and timing
- [ ] Call out current lot, price, and countdown clearly
- [ ] Watch for lag/audio issues every 10-15 minutes
- [ ] If issue appears: pause, explain briefly, recover calmly

## 5) Fast Recovery Plan (if something breaks)
- [ ] Stop host broadcast
- [ ] Wait 3-5 seconds
- [ ] Restart host broadcast
- [ ] If still broken, switch to backup host device
- [ ] Keep audience informed with one clear status update

## 6) Post-Live (after event)
- [ ] End broadcast cleanly
- [ ] Note issues and timestamps
- [ ] Save what worked / what failed
- [ ] Create top 3 fixes for next event

## Notes
- For first event reliability: stay RTC-only and avoid RTMP/OBS switching.
- Add OBS/RTMP in a future rehearsal once support confirms stream key API details.
